# 标签自动打标（Auto-Tag）— 设计 spec

- 日期：2026-06-25
- 状态：已通过 brainstorming，待出实现计划
- 所属 skill：pingcode-presales

## 1. 背景与动机
现有 `scripts/pingcode_tags.js` 只产出**标签名（字符串数组）**，既不在 PingCode 建标签、也不贴到工作项；`pingcode_api.js` 没有标签 API。本 feature 补上缺失的执行：**对一个已建项目，自动建若干标签并贴到对应工作项**，增强演示环境的可筛选性/业务感。

## 2. 目标与非目标
**目标**：独立操作——指定一个已建项目 → 扫内容自动提领域标签 → 在项目建标签 → 按关键词把标签贴到命中的工作项。
**非目标**：
- ❌ 不做 key=value 带值标签、不做自定义字段填值（本次只做 PingCode 普通标签 label）。
- ❌ 不绑进搭建主流程（独立脚本，按需对任意已建项目跑）。
- ❌ 不用 LLM 做匹配（关键词匹配，可解释、可批量）。

## 3. 已定决策（brainstorming 结论）
| 维度 | 决策 |
|------|------|
| 标签语义 | PingCode **标签(label)**，建标签 + 贴到工作项；"值"=哪些标签贴哪个工作项 |
| 标签来源 | **扫项目内容自动提**（高频领域词当标签） |
| 抽取方式 | **词典锚定**：项目内容 ∩ 领域词典（行业模板术语 + `pingcode_tags.js` 行业词典），频次≥N |
| 匹配逻辑 | **关键词**：工作项标题/描述含标签词 → 贴该标签 |
| 范围 | **独立脚本**，对已建项目跑，幂等 |

## 4. 架构
新增 `scripts/pingcode_tag_apply.js`，纯逻辑部分可独立单测：
```
认证 → 拉项目全部工作项
     → 抽标签(内容 ∩ 词典, 频次≥N, 按频降序封顶~12)
     → 建标签(项目级, 幂等复用)
     → 关键词匹配, 并发给命中工作项贴标(幂等跳过已贴)
     → 报告(建/复用数, 每标签贴数, 覆盖率, 失败)
```
用法：`node pingcode_tag_apply.js --client_id=.. --client_secret=.. --identifier=JLALED [--industry=汽车电子] [--min=2] [--max=12] [--dry]`

## 5. 数据流与匹配规则
- **领域词典**：合并「行业模板 `references/verticals/<行业>.md` 的术语行」+「`pingcode_tags.js` INDUSTRY_TERMS[行业]」。`--industry` 缺省时从项目名/描述推断或仅用通用词。
- **抽标签**：对每个词典词，统计其在所有工作项 `title + description` 中出现的工作项数；保留 **count ≥ min（默认2）**，按 count 降序取前 **max（默认12）** 个为标签集。
- **匹配贴标**：工作项 `title|description` 含某标签词（不区分大小写；支持别名映射，如 `ADB自适应远光`→`ADB`）即贴。一个工作项可多标签。空命中工作项不贴。

## 6. 技术未知（最大风险，实现前先探通）
`pingcode_api.js` 无标签 API，须现探：
1. **建标签**：候选 `POST /v1/project/tags`（或项目作用域端点），字段待探（name/color/project_id）。
2. **贴标签到工作项**：工作项对象含 `tags` 字段 → 候选 `PATCH /v1/project/work_items/{id} { tag_ids:[...] }` 或专用关联端点。
3. **探法**：API 直探；若被开放API限制（参照 api.md 测试用例关联坑），用「UI 建一个 → GET 反推字段」破解。**实现 Task 1 必须先把建/贴 API 探通并写进 `pingcode_api.js` 封装（createTag / attachTags）。**

## 7. 幂等 / 报告
- 建标签前 `GET 项目标签` → 同名复用，不重复建。
- 贴标前读工作项现有 `tags` → 已含则跳过。
- 贴标并发用 `batchCreateParallel(concurrency=10)`。
- 报告：标签集(建N/复用K)、每标签命中工作项数、贴标总数、覆盖率(被贴工作项/总数)、失败列表。

## 8. 测试
- **纯逻辑单测**（node:test）：给 mock 工作项数组 + 词典，断言 ①抽出的标签集（频次/封顶正确）②每个工作项的标签分配（关键词命中正确、别名生效、大小写无关、空命中不贴）。
- **API 验证**：在真项目（JLALED/睿恩）跑 `--dry` 预览，再实跑，核对 PingCode 里标签与贴标。

## 9. MVP 边界
- ✅ 词典锚定抽标签 + 关键词匹配 + 建/贴 API 封装 + 幂等 + 报告 + `--dry`
- ✅ 纯逻辑单测
- ❌ 不做 key=value 标签 / 自定义字段 / LLM 匹配 / 绑进搭建流程（均非目标）

## 10. 未决
- 标签 API 的确切端点/字段 → Task 1 探明后回填本 spec 与 `pingcode_api.js`。
- 别名映射表初版手列（ADB/UDS/配光…），后续可扩。
