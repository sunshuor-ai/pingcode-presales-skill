# 汽车电子 — 行业结构模板
<!-- synopsis: 汽车电子 / 车载 / 智能座舱 / ADAS / 感知系统 / 三电 / 毫米波雷达 / 芯片 / T-Box / 主机厂 / Tier1 / 域控 -->

## 概览
- 项目类型: hybrid（V模型 + 敏捷 融合；主机厂强调 ASPICE 合规）
- 阶段门（ASPICE V 模型，左下降→右上升）:
  - 系统线: SYS.1 利益相关方需求 → SYS.2 系统需求分析 → SYS.3 系统架构设计 →〔右〕SYS.4 系统集成与集成测试 → SYS.5 系统合格性测试
  - 软件线: SWE.1 软件需求分析 → SWE.2 软件架构设计 → SWE.3 详细设计与单元构建 →〔右〕SWE.4 单元验证 → SWE.5 软件集成与集成测试 → SWE.6 软件合格性测试
- **工作项骨架（需求驱动，非敏捷 epic/feature/story；完整 V 模型双侧，每个阶段都要有内容）**:
  - 左侧分解链：`阶段 → 需求:客户需求(SYS.1) → 需求:系统需求(SYS.2) → 子需求:系统架构(SYS.3) → 子需求:软件需求/架构(SWE.1/2) → 子需求:详细设计(SWE.3) → 任务:软件单元实现`
  - 右侧验证：`任务:软件单元验证(SWE.4) / 软件集成测试(SWE.5) / 软件合格性测试(SWE.6) / 系统集成测试(SYS.4) / 系统合格性测试(SYS.5) / 量产验证`（挂对应需求，双向追溯）
  - 里程碑（用 `parent_id` 挂阶段）：立项评审 / 设计冻结(A样) / DV通过 / 量产放行(SOP)
  —— 全部用【需求】逐级条目化（子需求嵌套，跨阶段父子），实现/验证用【任务】，缺陷挂需求；自定义需求 ID。**不要用 史诗/特性/用户故事**。**G1 概念立项、G7 系统验证/量产不能空**。
- 测试维度: 单元验证(SWE.4) / 软件集成(SWE.5) / 软件合格性(SWE.6) / 系统集成(SYS.4) / 系统合格性(SYS.5)；HIL/SIL、EMC、功能安全(ISO 26262)、回归测试
- 双向追溯: **纵向**=需求多级分解树(parent_id, 开放API已可建)。**横向 需求↔TestHub用例**=开放API只认用户故事(见 api.md #37)，需求驱动项目走 Web自动化 或 交付文档给覆盖对照表；TestHub 库/套件/用例本身开放API可建并按验证维度组织。
- 标准: ASPICE（主机厂准入）/ ISO 26262 功能安全 / IATF 16949
- 术语: ECU, T-Box, ASIL, DFMEA, AUTOSAR, OTA, FAE, 主机厂, Tier1/Tier2, DOORS, 毫米波雷达, 智能座舱, 三电, 配光, ADB, 矩阵, 远光, 近光, UDS, EMC, 诊断, CAN, LIN, 故障检测, 功能安全, 驱动, 域控
- PingCode 短板→补位（演示抓手）: SUP.10 变更请求（官方标注不满足）、强追溯/基线/覆盖率统计短板 → 用自定义类型补
- 客户关注点(实战增量): ⟵ 你叠加（谈单时客户真正在意的点）

## 专属类型与字段
```yaml
types:
  - name: 变更请求
    group: 事务
    fields:
      - { name: 变更类型, kind: 单选, options: [需求变更, 设计变更, 架构变更, 工艺变更, 软件变更] }
      - { name: 影响分析, kind: 多行文本 }
      - { name: 受影响基线, kind: 单行文本 }
      - { name: 评审结论, kind: 单选, options: [通过, 有条件通过, 不通过] }
  - name: 技术评审
    group: 事务
    fields:
      - { name: 评审类型, kind: 单选, options: [需求评审, 架构评审, 设计评审, 代码评审, 测试评审] }
      - { name: 评审结论, kind: 单选, options: [通过, 有条件通过, 不通过] }
      - { name: 问题清单, kind: 多行文本 }
      - { name: 评审日期, kind: 日期 }
  - name: 功能安全需求
    group: 需求
    fields:
      - { name: ASIL等级, kind: 单选, options: [QM, A, B, C, D] }
      - { name: 安全目标, kind: 多行文本 }
      - { name: 关联危害, kind: 单行文本 }
  - name: 风险项
    group: 事务
    fields:
      - { name: 风险等级, kind: 单选, options: [高, 中, 低] }
      - { name: 风险类型, kind: 单选, options: [技术风险, 进度风险, 合规风险, 供应链风险] }
      - { name: 应对措施, kind: 多行文本 }
  - name: 基线评审
    group: 任务
    fields:
      - { name: 基线类型, kind: 单选, options: [需求基线, 设计基线, 测试基线, 发布基线] }
      - { name: 基线版本, kind: 单行文本 }
      - { name: 评审结论, kind: 单选, options: [通过, 不通过] }
```
