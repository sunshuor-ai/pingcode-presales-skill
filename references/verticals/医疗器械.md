# 医疗器械 — 行业结构模板
<!-- synopsis: 医疗器械 / 医疗软件 / SaMD / 体外诊断 / IVD / 有源器械 / 植入器械 / 影像 -->

## 概览
- 项目类型: hybrid（IEC 62304 软件生命周期 + 设计控制；阶段门为主、部分敏捷）
- 阶段门（IEC 62304 + 设计控制 DHF）:
  设计与开发策划 → 软件需求分析 → 软件架构设计 → 软件详细设计 → 单元实现与验证 → 软件集成与集成测试 → 软件系统测试 → 设计验证与确认(V&V) → 设计转换/放行
- 工作项层级要点: 需求-风险-验证 追溯矩阵；设计历史文档(DHF)；软件安全等级 A/B/C 分级
- 测试维度: 单元/集成/系统测试、设计验证(Verification)、设计确认(Validation)、可用性测试(IEC 62366)
- 标准: IEC 62304(软件生命周期) / ISO 14971(风险管理) / ISO 13485(质量体系) / IEC 62366(可用性) / FDA 21 CFR 820 / NMPA
- 术语: SaMD, DHF, DMR, V&V, 安全等级A/B/C, 风险受益, CAPA, IVD, UDI, 召回
- PingCode 短板→补位（演示抓手）: 风险管理矩阵、可追溯报告、变更控制 → 用自定义类型补
- 客户关注点(实战增量): ⟵ 你叠加

## 专属类型与字段
```yaml
types:
  - name: 风险项
    group: 事务
    fields:
      - { name: 危害, kind: 单行文本 }
      - { name: 严重度, kind: 单选, options: [可忽略, 轻微, 严重, 危重, 灾难性] }
      - { name: 发生概率, kind: 单选, options: [极少, 偶尔, 有时, 可能, 频繁] }
      - { name: 风险控制措施, kind: 多行文本 }
      - { name: 剩余风险可接受性, kind: 单选, options: [可接受, 需监控, 不可接受] }
  - name: 设计变更控制
    group: 事务
    fields:
      - { name: 变更类型, kind: 单选, options: [需求变更, 设计变更, 风险变更, 标签变更] }
      - { name: 影响分析, kind: 多行文本 }
      - { name: 是否需重新验证, kind: 单选, options: [是, 否] }
      - { name: 审批结论, kind: 单选, options: [通过, 不通过] }
  - name: 设计评审
    group: 事务
    fields:
      - { name: 评审阶段, kind: 单选, options: [需求评审, 架构评审, 设计评审, V&V评审, 放行评审] }
      - { name: 评审结论, kind: 单选, options: [通过, 有条件通过, 不通过] }
      - { name: 问题清单, kind: 多行文本 }
      - { name: 评审日期, kind: 日期 }
  - name: 验证与确认
    group: 任务
    fields:
      - { name: 类型, kind: 单选, options: [设计验证Verification, 设计确认Validation, 可用性测试] }
      - { name: 验收准则, kind: 多行文本 }
      - { name: 结论, kind: 单选, options: [通过, 失败, 待定] }
```
